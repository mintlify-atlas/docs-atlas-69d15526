Hyrax is a [Ruby on Rails Engine](https://guides.rubyonrails.org/engines.html) built by the [Samvera community](https://samvera.org). It gives you a head start building digital repository applications — providing works, collections, configurable deposit workflows, flexible metadata, IIIF support, and an administrative dashboard out of the box. You mount Hyrax inside your own Rails application and customize it to fit your institution's needs.

## What Hyrax provides

Hyrax bundles everything you need to operate a digital repository:

- **Work types** — Generate repository object types on demand using Rails generators. Each work type gets its own model, form, indexer, and controller.
- **Deposit workflows** — Route submissions through configurable Sipity-based approval workflows with multiple reviewers and states.
- **Flexible metadata** — Define metadata schemas via YAML files or the M3 profile UI without touching Ruby code.
- **IIIF support** — Serve IIIF manifests and integrate with Universal Viewer or your own image server.
- **Collections** — Organize works into nested, typed collections with granular access controls.
- **Admin dashboard** — Toggle features, manage users, configure workflows, and view analytics from a built-in UI.
- **Background jobs** — Process file ingestion, derivative generation, characterization, and fixity checking asynchronously via ActiveJob.

## Get started

<Steps>
  <Step title="Install prerequisites">
    Set up Ruby 3.3+, Rails 7.2, Solr, Fedora (or PostgreSQL via Valkyrie), Redis, ImageMagick, FITS, and LibreOffice.
  </Step>
  <Step title="Generate your application">
    Use the Hyrax application template to scaffold a new Rails app with Hyrax pre-configured:
    ```bash
    HYRAX_SKIP_WINGS=true rails _7.2.2.1_ new my_app --database=postgresql \
      -m https://raw.githubusercontent.com/samvera/hyrax/hyrax-v5.2.0/template.rb
    ```
  </Step>
  <Step title="Start services and run migrations">
    Start your backend services (Solr, Postgres, Redis) then run migrations and seed data:
    ```bash
    rails db:migrate
    rails db:seed
    ```
  </Step>
  <Step title="Generate a work type and launch">
    Create at least one work type, then start the Rails server:
    ```bash
    rails generate hyrax:work_resource GenericWork
    rails server
    ```
    Browse to [localhost:3000](http://localhost:3000) to see your repository.
  </Step>
</Steps>

<Note>
  Hyrax is a Rails Engine — not a standalone web application. You must mount it inside your own Rails app. The application template handles this automatically when you generate a new app.
</Note>
