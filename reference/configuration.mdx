description: "Complete reference for the Hyrax.config block: analytics, file handling, IIIF, local storage, Valkyrie adapters, and all other engine options."
---

Hyrax is configured through an initializer that yields a `Hyrax::Configuration` instance. All options have sensible defaults, so you only need to set the values that differ from those defaults. The recommended pattern is to edit `config/initializers/hyrax.rb` (generated by `rails generate hyrax:install`).

```ruby
# config/initializers/hyrax.rb
Hyrax.config do |config|
  config.analytics            = true
  config.analytics_provider   = 'ga4'
  config.work_requires_files  = false
  config.iiif_image_server    = true
  config.derivatives_path     = Rails.root.join('tmp', 'derivatives')
end
```

## Analytics

Options that control whether Hyrax collects page-view and download statistics and which third-party analytics provider is used.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `analytics` | Boolean | `false` | Master switch for analytics collection. Can be set with `HYRAX_ANALYTICS=true`. |
| `analytics_provider` | String | `'ga4'` | Analytics provider to use. Supported values: `'ga4'`, `'matomo'`. Reads `HYRAX_ANALYTICS_PROVIDER`. |
| `analytics_reporting` | Boolean | `false` | Show analytic reports in the dashboard. Requires `HYRAX_ANALYTICS_REPORTING` plus the Google OAuth environment variables to be set. |
| `analytics_start_date` | String | One year ago | Date (YYYY-MM-DD) from which "all-time" metrics are computed. Reads `ANALYTICS_START_DATE`. |

```ruby
Hyrax.config do |config|
  config.analytics           = true
  config.analytics_provider  = 'ga4'
  config.analytics_reporting = true
  config.analytics_start_date = '2020-01-01'
end
```

## Files

Options that govern how uploaded files are characterised, derivative generation, fixity checking, and related file-handling behaviour.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `characterization_service` | Class | `Hyrax::Characterization::ValkyrieCharacterizationService` | Service class used to extract technical metadata from files. Must respond to `.run`. |
| `characterization_options` | Hash | `{ ch12n_tool: :fits }` | Options forwarded to the characterization service. Set `CH12N_TOOL` env var to change the tool (e.g. `fits_servlet`). |
| `characterization_proxy` | Symbol | `:original_file` | Which attached file is used for MIME-type resolution. |
| `derivative_services` | Array | `[Hyrax::FileSetDerivativesService]` | Ordered list of derivative service classes. The first class where `#valid?` returns true will handle generation. |
| `max_days_between_fixity_checks` | Integer | `7` | Number of days between scheduled fixity audits on stored files. |
| `work_requires_files` | Boolean | `true` | When `true`, depositing a work without at least one attached file is blocked. |
| `solr_rows_per_request` | Integer | `1000` | Maximum Solr rows returned per internal query. |
| `solr_max_results` | Integer | `10000` | Hard cap on total Solr results retrieved for operations such as batch exports. |
| `extract_full_text` | Boolean | `true` | When `true`, full-text content is extracted from documents and indexed in Solr. |

```ruby
Hyrax.config do |config|
  config.characterization_options = { ch12n_tool: :fits_servlet }
  config.derivative_services      = [MyApp::CustomDerivativeService, Hyrax::FileSetDerivativesService]
  config.max_days_between_fixity_checks = 14
  config.work_requires_files      = false
end
```

## Identifiers

Options controlling the NOID (Nice Opaque Identifier) minting strategy.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `enable_noids` | Boolean | `true` | Enable database-backed NOID minting for new objects. |
| `noid_template` | String | `'.reeddeeddk'` | Noid template string passed to the minter. |
| `noid_minter_class` | Class | `Noid::Rails::Minter::Db` | Minter class to use. Must conform to the Noid::Rails minter interface. |
| `minter_statefile` | String | `'/tmp/minter-state'` | Path to the minter state file when using a file-based minter. |

## IIIF

Options for the embedded IIIF image server, manifest generation, and the AV viewer.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `iiif_image_server` | Boolean | `false` | Enable the IIIF image service. Required for the IIIF-enabled show page. |
| `iiif_image_url_builder` | Lambda | download URL fallback | A callable that returns the full IIIF image URL for a given `file_id`, `base_url`, `size`, and `format`. |
| `iiif_info_url_builder` | Lambda | `-> { '' }` | A callable that returns the IIIF `info.json` URL for a file. |
| `iiif_image_compliance_level_uri` | String | `'http://iiif.io/api/image/2/level2.json'` | IIIF compliance level URI advertised in manifests. |
| `iiif_image_size_default` | String | `'600,'` | Default IIIF size parameter appended to image requests. |
| `iiif_manifest_cache_duration` | Integer | `2592000` (30 days) | How long (in seconds) generated IIIF manifests are cached. |
| `iiif_manifest_factory` | Class | `IIIFManifest::ManifestFactory` | Class responsible for building IIIF manifests. |
| `iiif_av_viewer` | Symbol | `:universal_viewer` | Partial name of the viewer rendered for audio/video items. |
| `rendering_predicate` | RDF::URI | `RDF::Vocab::DC.hasFormat` | RDF predicate used for IIIF `rendering` entries. |

```ruby
Hyrax.config do |config|
  config.iiif_image_server = true

  config.iiif_image_url_builder = lambda do |file_id, base_url, size, format|
    "#{base_url}/iiif/2/#{CGI.escape(file_id)}/full/#{size}/0/default.#{format}"
  end

  config.iiif_info_url_builder = lambda do |file_id, base_url|
    "#{base_url}/iiif/2/#{CGI.escape(file_id)}/info.json"
  end

  config.iiif_manifest_cache_duration = 7.days.to_i
  config.iiif_av_viewer               = :universal_viewer
end
```

## Local Storage

Paths on the local filesystem where Hyrax stores working copies of files. All of these can also be set via environment variables.

| Option | Type | Default | Env var |
|--------|------|---------|---------|
| `derivatives_path` | Pathname | `Rails.root/tmp/derivatives` | `HYRAX_DERIVATIVES_PATH` |
| `working_path` | Pathname | `Rails.root/tmp/uploads` | `HYRAX_UPLOAD_PATH` |
| `branding_path` | Pathname | `Rails.root/public/branding` | `HYRAX_BRANDING_PATH` |
| `bagit_dir` | String | `'tmp/descriptions'` | — |

```ruby
Hyrax.config do |config|
  config.derivatives_path = Rails.root.join('shared', 'derivatives')
  config.working_path     = Rails.root.join('shared', 'uploads')
  config.branding_path    = Rails.root.join('public', 'branding')
end
```

## System Dependencies

Paths and flags for external tools Hyrax shells out to.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `enable_ffmpeg` | Boolean | `false` | Enable FFmpeg-based video derivative generation. |
| `ffmpeg_path` | String | `'ffmpeg'` | Path to the `ffmpeg` binary. Reads `HYRAX_FFMPEG_PATH`. |
| `fits_path` | String | `'fits.sh'` | Path to the FITS characterization script. Reads `HYRAX_FITS_PATH`. |
| `virus_scanner` | Class | `Hyrax::VirusScanner` | Virus-scanning service class. Must respond to `.infected?`. |
| `libreoffice_path` | String | `'soffice'` | Path to the LibreOffice `soffice` binary used for office document derivatives. |

```ruby
Hyrax.config do |config|
  config.enable_ffmpeg = true
  config.ffmpeg_path   = '/usr/local/bin/ffmpeg'
  config.fits_path     = '/opt/fits/fits.sh'
  config.virus_scanner = Hyrax::VirusScanner  # swap for ClamAV wrapper if needed
end
```

## Theme and UI

Options that control the appearance and interactive behaviour of the Hyrax interface.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `banner_image` | String | Remote JPEG URL | URL of the hero image shown on the homepage. |
| `breadcrumb_builder` | Class | `Hyrax::BootstrapBreadcrumbsBuilder` | Class responsible for rendering breadcrumb navigation. |
| `display_share_button_when_not_logged_in` | Boolean | `true` | Show the "Share your work" call-to-action on the front page for unauthenticated users. |
| `display_media_download_link` | Boolean | `true` | Show the download link on media file-set pages. |
| `citations` | Boolean | `false` | Enable formatted citation generation on work show pages. |
| `realtime_notifications` | Boolean | `true` | Enable ActionCable-powered real-time dashboard notifications. Automatically coerced to `false` under Passenger + Apache. |
| `max_notifications_for_dashboard` | Integer | `5` | Number of recent notifications shown in the dashboard notification area. |
| `contact_email` | String | `'repo-admin@example.org'` | Destination address for contact form submissions. |
| `show_work_item_rows` | Integer | `10` | Number of member items shown per page on the work show view. |

## Valkyrie

Options for toggling the Wings compatibility adapter and configuring Valkyrie persistence and indexing.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `use_valkyrie` | Boolean | `false` | Use Valkyrie storage features for new deposits. Reads `HYRAX_VALKYRIE`. |
| `disable_wings` | Boolean | `false` | Disable the Wings ActiveFedora-compatibility layer entirely. Reads `HYRAX_SKIP_WINGS`. Forces `use_valkyrie` to `true`. |
| `disable_freyja` | Boolean | `false` | Disable the Freyja adapter. Reads `HYRAX_SKIP_FREYJA`. |
| `disable_frigg` | Boolean | `false` | Disable the Frigg adapter. Reads `HYRAX_SKIP_FRIGG`. |
| `index_adapter` | Symbol / Adapter | `:null_index` | The Valkyrie indexing adapter registered with `Valkyrie::IndexingAdapter`. |
| `query_index_from_valkyrie` | Boolean | `false` | When `true`, read queries are routed through the Valkyrie index adapter instead of Solr directly. |
| `pcdm_object_indexer_builder` | Lambda | auto-builder | Lambda that returns an indexer class for a given work model class. |

```ruby
Hyrax.config do |config|
  config.use_valkyrie         = true
  config.index_adapter        = :solr_index
  config.query_index_from_valkyrie = true
end
```

## Miscellaneous

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `ingest_queue_name` | Symbol | `:default` | ActiveJob queue used for ingest-style jobs. Reads `HYRAX_ACTIVE_JOB_QUEUE`. |
| `redis_namespace` | String | `'hyrax'` | Redis key namespace for cache and pub/sub operations. Reads `HYRAX_REDIS_NAMESPACE`. |
| `default_active_workflow_name` | String | `'default'` | Name of the Sipity workflow activated when a new admin set is created. |
| `active_deposit_agreement_acceptance` | Boolean | `true` | Require depositors to accept the deposit agreement before uploading. |
| `system_user_key` | String | `'systemuser@example.com'` | Email key of the internal system user used for automated operations. |
| `batch_user_key` | String | `'batchuser@example.com'` | Email key of the user attributed to batch import operations. |
| `lock_retry_count` | Integer | `600` | Number of attempts to acquire a distributed lock before raising `UnableToAcquireLockError`. |
| `lock_time_to_live` | Integer | `60000` | How long a distributed lock is held, in milliseconds. |
| `range_for_number_of_results_to_display_per_page` | Array | `[10, 20, 50, 100]` | Options presented in the per-page selector on search result pages. |
