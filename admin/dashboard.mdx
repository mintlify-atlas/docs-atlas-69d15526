title: "Using the Hyrax Administrative Dashboard"

The administrative dashboard is the central hub for managing a Hyrax repository. It groups tools for content management, workflow configuration, user administration, and site appearance under a single interface, accessible at `/dashboard`. Only users assigned the admin role can see the full set of administrative controls.

## Accessing the dashboard

Navigate to `/dashboard` after signing in. Users who hold the admin role see the complete sidebar, including all administrative sections. Registered users without the admin role see a reduced view that only contains their own works, collections, and profile.

<Note>
  If a user visits `/dashboard` without the admin role, Hyrax renders the personal dashboard with their own works and transfers. Administrative sections — Features, Workflows, Appearance, and others — are hidden and access-controlled at the controller level.
</Note>

## Granting admin access

Hyrax resolves role membership through your application's role map file. Add an email address to the `admin` key to promote a user.

<Steps>
  <Step title="Open the role map">
    Edit `config/role_map.yml` in your application. If the file does not exist, create it.

    ```yaml config/role_map.yml
    development:
      admin:
        - admin@example.org

    production:
      admin:
        - repository.admin@institution.edu
        - another.admin@institution.edu
    ```
  </Step>
  <Step title="Restart the application">
    Role map changes are read at startup. Restart your Rails server or application process for the new assignment to take effect.

    ```bash
    bundle exec rails server
    ```
  </Step>
  <Step title="Verify access">
    Sign in as the newly promoted user and visit `/dashboard`. The full administrative sidebar should be visible.
  </Step>
</Steps>

## Dashboard sections

The dashboard sidebar organises tools into two broad areas: your personal content and site-wide administration.

### Content management

| Section | Route | Purpose |
|---|---|---|
| Works | `/dashboard/works` | Browse and manage all works in the repository |
| Collections | `/dashboard/collections` | Browse and manage all collections |
| Admin Sets | `/admin/admin_sets` | Create and configure administrative containers |

### Administration

| Section | Route | Purpose |
|---|---|---|
| Users | `/admin/users` | List registered accounts |
| Workflows | `/admin/workflows` | Configure Sipity approval workflows |
| Features | `/admin/features` | Toggle optional features via Flipflop |
| Analytics | `/admin/analytics` | View work and collection usage reports |
| Appearance | `/admin/appearance` | Customise banner images and colour theme |
| Metadata Profiles | `/admin/metadata_profiles` | Import and manage M3 metadata profiles |

## Activity feed

The dashboard homepage displays a recent activity feed drawn from repository events. The feed defaults to showing activity from the last 24 hours. You can adjust the look-back window in your Hyrax initialiser.

```ruby config/initializers/hyrax.rb
Hyrax.config do |config|
  # Show activity from the last 48 hours (value is in seconds)
  config.activity_to_show_default_seconds_since_now = 48 * 60 * 60

  # Limit the number of notifications shown in the dashboard panel
  config.max_notifications_for_dashboard = 10
end
```

The activity feed is separate from the notifications panel. Notifications are user-scoped messages generated by workflow actions and other events.
